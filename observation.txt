The inventory number changes each run because multiple bot processes simultaneously
read the same inventory value before any of them writes the updated value back.
This is a TOC-TOU (Time-of-Check to Time-of-Use) race condition — the OS scheduler
runs processes in a non-deterministic order, so different numbers of bots "win"
the race each time, causing inconsistent results.
